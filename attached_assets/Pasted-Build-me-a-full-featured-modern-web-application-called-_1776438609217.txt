Build me a full-featured, modern web application called "ScriptureLive AI" — an AI-powered Bible and church worship presentation platform inspired by LogosAI (thelogosai.com). The app should include the following features and pages:

🎯 Core Features:
Real-Time Scripture Detection — Use speech-to-text (Web Speech API or similar) to listen to a live sermon/speaker and automatically detect when Bible verses are mentioned. Instantly display the full verse text on-screen with sub-2-second latency.
Bible Verse Lookup & Display — A searchable Bible database (use a free Bible API like bible-api.com or scripture.api.bible) where users can search by book, chapter, verse, keyword, or topic. Display verses in a clean, large-font, projector-ready format.
Multiple Bible Translations — Support at least KJV, NIV, ESV, and NKJV translations with a toggle/dropdown selector.
AI-Powered Slide Generator — Users can type or paste a sermon topic, Bible passage, or theme, and the AI generates beautiful presentation slides (with titles, subtitles, verse text, and backgrounds). Allow export to PDF or image.
Worship Lyrics Manager — A module where users can:
Paste raw song lyrics and have the AI auto-structure them into Verse, Chorus, Bridge, etc.
Search a cloud lyrics database for hymns and modern worship songs.
Display lyrics in a projector-ready full-screen mode with smooth transitions.
Live Presenter Mode — A dual-screen system: one screen for the operator (preview + controls), and a second output screen for the congregation (clean display). Include next-slide preview.

Sermon Notes & Outline Builder — A simple rich-text editor where pastors can build sermon outlines, link Bible references (which auto-populate the verse text), and export as PDF.
Import/Export — Allow importing song libraries from common formats (XML, plain text). Export presentations as PDF, PPTX, or image slides.

Modern, clean, dark-themed UI (suitable for church media teams working in low-light environments).
Responsive design — works on desktop, tablet, and mobile.
